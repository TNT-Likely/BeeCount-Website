# 多币种(主币种折算)

> ✨ **v3.5.0 新增**;自建云用户需要 BeeCount Cloud ≥ 1.5.0。

手里同时有人民币、美元、日元账户,资产页却只能按币种分组、各看各的,加不到一起?多币种功能让你**设置一个主币种**,再为其它币种**维护一份汇率**,资产页就能把所有账户折算成主币种,给出一个统一的净资产数字。

:::info 只有一种币种?无需关心
全部账户都是同一币种时,行为和旧版本完全一致——不显示折算、不需要设主币种,这个功能对你是隐形的。
:::

:::info 支持全量国际货币(v3.5.2 起)
可选币种已覆盖全量 ISO 4217 共 **151 种**(新增肯尼亚先令 KES、中非 / 西非法郎 XAF/XOF 等非洲、中东、拉美币种)。主流币种显示本地化名称,其余显示标准英文名;暂不支持自定义币种。
:::

## 账本内多币种交易(v3.6.0)

v3.6.0 起,多币种从下面讲的「资产页折算」进一步扩展到了**账本里的每一笔交易**:

- 每个账本有一个**主币种**(即原来的「账本币种」,在账本编辑页设置)——账本的月度收支、分类汇总、预算用量等统计都以它为口径
- 记账时可以**选择这笔交易的币种**:选了账户就跟随账户币种,没选账户时可手动选;选外币时,金额下方实时显示按汇率折算成账本主币种后的金额
- 账本的各项统计会把外币交易**按汇率自动折算到主币种**再汇总,同一账本里不同币种的交易终于能加到一起
- 每笔交易始终**原样保存你实际支付的原币金额**;折算快照在记账当时按汇率生成后**固定**,之后再改汇率不会改动历史交易已记的折算

:::info 「账本主币种」和「用户主币种」是两个层面
本页下半部分的「用户主币种」是资产页把不同币种**账户**跨账本折算成一个净资产数字;这里的「账本主币种」是一个账本内不同币种**交易**的统计折算。两者各管一摊,但**共用你维护的同一份汇率**。单币种账本完全不受影响。
:::

## 设置主币种

1. 进入「我的」→ **多币种 / 汇率设置**(或资产页折算汇总旁的设置入口)
2. 选择**主币种**——之后资产页的统一汇总、净值趋势都以它为口径
3. 主币种设置随云同步推送到所有设备

主币种是一个**用户级**设置(对你的所有账本生效),不是单账本设置。

## 管理汇率

在同一页面维护**其它币种 → 主币种**的汇率:

1. 点击要添加/修改的币种
2. 填入相对主币种的汇率(例如主币种是 CNY,给 USD 填"1 USD = 7.x CNY")
3. 保存后立即生效,资产页重新折算

- 汇率是**手动维护**的固定值,改了主币种后需要重新核对各币种汇率
- Web 端(自建云)同样可以编辑汇率,改完秒级同步到手机
- 自建云提供**汇率代理**,后续可在服务端侧获取参考汇率(详见服务端文档),App / Web 编辑的手动汇率始终优先

## 资产页折算汇总

设好主币种和汇率后,资产页顶部会多出**按主币种折算的统一汇总**:

- 总资产 / 总负债 / 净资产都换算成主币种相加
- 原有的**按币种分组明细**保持不变,折算汇总是叠加在上面的一层总览
- [净值趋势](./net-worth-trend.md)图同样以主币种为口径绘制

## 注意事项

- **必须先设主币种**:没设主币种时,资产页维持按币种分组、不做折算
- **缺汇率的币种**:如果某个币种没有维护到主币种的汇率,该币种的账户**无法被折算**,会从主币种汇总中**排除**(分组明细里仍照常显示)。建议把用到的每个币种都补上汇率
- **手动汇率、非实时**:汇率是你填的固定值,不会自动按市场波动更新;大额或汇率剧烈波动时记得手动校准
- **折算只影响展示**:每笔交易、每个账户的原始币种与金额都原样保存,折算只发生在资产页的汇总展示层,改主币种/汇率不会改动任何账户余额

## 版本要求

| 功能 | App | BeeCount Cloud(自建云) |
|---|---|---|
| 资产页折算(用户主币种) | ≥ 3.5.0 | ≥ 1.5.0 |
| 账本内多币种交易(账本主币种) | ≥ 3.6.0 | ≥ 1.6.0 |

- **资产折算(v3.5.0)**:主币种与汇率以**用户级**维度同步(user-global projection),无数据库迁移;老服务端不识别主币种设置会表现为"折算设置不跨设备",升级 Cloud 到 1.5.0 即对齐。
- **账本多币种(v3.6.0)**:交易表新增币种 / 折算两列,**含数据库 schema 迁移**(随镜像启动自动执行 + 历史回填);老服务端不识别这两列,新 App 记的外币交易在 Web / 跨设备会折算丢失、账本统计按原币裸加,升级 Cloud 到 1.6.0 即对齐。请**先升级服务端、再升 App**。
