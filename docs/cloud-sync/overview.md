# 云同步概述

蜜蜂记账支持多种云同步方式，数据存储灵活可控。

## 为什么需要云同步？

- **数据备份** - 防止数据丢失
- **换机迁移** - 新设备快速恢复数据
- **设备切换** - 在不同设备间切换使用（需按正确步骤操作）

## 支持的同步方式

| 方式 | 适合人群 | 难度 |
|------|---------|------|
| [iCloud](./icloud) | iOS 用户 | ⭐ 最简单 |
| [Supabase](./supabase) | 想要免费云数据库 | ⭐⭐ 简单 |
| [WebDAV](./webdav) | 使用坚果云/群晖 | ⭐⭐ 简单 |
| [S3](./s3) | 技术用户 | ⭐⭐⭐ 中等 |

## 数据安全

- 所有数据都存储在**你自己的**云存储中
- 蜜蜂记账不收集、不存储任何用户数据
- 你可以随时导出、迁移数据

## 选择建议

- **iOS 用户**：优先使用 iCloud，零配置
- **Android 用户**：推荐 Supabase 或 WebDAV
- **技术用户**：可以自建 S3 或 WebDAV 服务

## 多设备使用说明

:::warning 重要提示

当前暂不支持多设备协同编辑。如需在多台设备间切换使用，请按以下步骤操作：

:::

### 正确的设备切换步骤

假设你想从设备 A 切换到设备 B：

1. **在设备 A 上**
   - 确保数据已同步到云端
   - 可以在云同步设置页面查看最后同步时间

2. **在设备 B 上**
   - 删除所有本地账本（或清空应用数据）
   - 配置相同的云同步服务
   - 从云端下载数据

3. **注意事项**
   - ⚠️ 切换前务必确认设备 A 数据已上传
   - ⚠️ 设备 B 必须先清空本地数据再下载
   - ⚠️ 不要在两台设备上同时编辑数据

### 为什么不支持多设备协同？

目前的云同步方案基于文件同步，不支持实时的冲突解决。如果多设备同时编辑，可能导致数据冲突或丢失。

### 未来计划

我们正在考虑升级云同步架构，支持真正的多设备实时协同。在此之前，请严格按照上述步骤切换设备。
