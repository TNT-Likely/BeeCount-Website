# 账单标记

> ✨ **v3.5.1 新增**;云模式下"按标记过滤收支/预算"由服务端提供,自建云用户需要 BeeCount Cloud ≥ 1.5.1(本地模式与单机统计始终正确,不依赖服务端)。

有些账记是要记的(账户余额得对),但不该算进"这个月花了多少"或"预算还剩多少"。比如替朋友垫付的一笔,或者一次突发的意外支出。账单标记让你给单笔交易打上标记,**把它从统计或预算里摘出来,但账还在**。

每笔交易有两个**完全独立**的开关,可任意组合:

## 不计入收支

打上后,这笔**不进入收支统计**——收支图表、月/年汇总、以及账户维度的收支数字都会把它剔除。

但它**仍然计入账户余额、净资产和资产趋势**,也**仍然显示在账单列表里**(带一个小标签)。

**典型场景:替朋友垫付。** 把"垫付的支出"和"朋友还款的收入"都标上「不计入收支」,收支统计就不会被这一进一出虚高,而账户余额因为一进一出本来就自然轧平了。

## 不计入预算

打上后,这笔**不占用预算额度**——预算的"已用/剩余"不把它算进去。

**典型场景:突发意外支出。** 临时多花的一笔不想打乱原有的预算规划,标上「不计入预算」即可,预算进度照常反映日常开销。

## 两者独立

这两个开关互不影响,可以任意组合:

| 组合 | 效果 |
|------|------|
| 仅「不计入收支」 | 不进收支统计,但仍占预算 |
| 仅「不计入预算」 | 不占预算,但仍进收支统计 |
| 两个都开 | 既不进收支统计,也不占预算 |
| 都不开 | 普通交易 |

## 关键:不影响余额与净资产

这是理解账单标记最重要的一点:**无论怎么标记,账户余额、净资产、资产趋势都不受影响。** 标记只改变"它算不算进收支统计 / 预算",不会让这笔钱凭空消失。所以垫付场景下账户余额永远是对的。

## 在哪设置

记账面板里,**附件图标后面有一个小旗标图标**,点击弹出对话框,在里面打开/关闭这两个开关。

开关按交易类型显示:

- **不计入收支**:收入、支出都可见
- **不计入预算**:仅支出可见
- **转账**:不显示标记入口

## 账单列表里的标识

已标记的交易会在账单列表里显示一个**小标签**,一眼就能认出哪些笔被摘出了统计或预算。

## 自建云用户须知

- **本地模式 / 单机统计**:始终按标记正确过滤,不依赖服务端。
- **云模式**:服务端的收支/预算汇总按标记过滤,需要 BeeCount Cloud ≥ 1.5.1。升级顺序**先服务端、后 App**,详见 [BeeCount Cloud 版本升级](../cloud-sync/beecount-cloud.md)。
